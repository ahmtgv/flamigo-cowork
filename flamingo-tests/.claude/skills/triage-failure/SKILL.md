# Разбор падения

1. Открой трейс: `npx playwright show-trace test-results/<...>/trace.zip`.
2. Классифицируй:
   - **Настоящий баг** — воспроизводится стабильно → чинить продукт.
   - **Флак** — падает через раз → см. п.3.
   - **Инфраструктура** — таймаут сети, недоступен stage → чинить окружение.
3. Частые причины флака в этом проекте:
   - ассерт на float вместо дискретного состояния (SEduM) → переписать;
   - гонка: нет ожидания появления элемента → `expect(...).toBeVisible()`;
   - тест зависит от данных другого теста → сделать изолированным;
   - медленная инициализация камеры → увеличить таймаут ПЕРВОГО ожидания.
4. **Если падает privacy-тест — это инцидент, а не флак.** Не ослаблять,
   не помечать `@quarantine`. Эскалировать.
5. Флак чинится в тот же день либо помечается `@quarantine` с заведённой задачей.
   Молча отключать нельзя.

# Конвенции тестирования Flamingo

Правила обязательны. Нарушение = тест не проходит ревью.

## Селекторы
- **Только** `getByTestId` / `getByRole`. Никогда CSS, никогда XPath.
- Новый testid сначала добавляется в `tests/config/flamingo.ts`, потом используется.
- В `.spec.ts` не должно быть ни одной строковой константы селектора.

## SEduM
- Ассерты **только на дискретное состояние** и UI-последствие.
- **Никогда** не ассертить float (`yaw === 17.3`) — гарантированный флак.
- Агрегат по клипу («≥90% кадров ATTENTIVE»), не пер-кадровые проверки.
- Каждый новый тест SEduM обязан иметь видео-фикстуру.

## Приватность
- Тесты помечаются `@privacy`.
- **Их нельзя ослаблять — только усиливать.** Ослабление privacy-теста
  требует явного согласования, это юридический контур, а не технический.
- Новый исходящий payload SEduM → сначала обновить `rawBiometricMarkers`
  и `allowedSedumEndpoints`, потом писать код.

## Права доступа
- Каждый новый эндпоинт → строка в `permissionMatrix`. Без исключений.
- Данные несовершеннолетних: отказ не должен «протекать» чужими идентификаторами.

## Общее
- Аутентификация — через `storageState`, не UI-логин в тесте.
- Никаких `waitForTimeout`, кроме privacy-наблюдения (там это осознанно).
- Тест изолирован: сам создаёт и убирает данные.
- Сообщение об ошибке должно быть понятно без чтения кода теста.
- Готов = 20 прогонов подряд зелёные (`npm run test:flake`).

## Что НЕ делать
- Не отключать упавший тест молча. Либо чинить, либо `@quarantine` + задача.
- Не ассертить FPS и тайминги в CI — раннеры шумные.
- Не гонять vision-пайплайн на GPU-делегате в CI (недетерминирован) — только CPU/XNNPACK.

# FrameSource — seam для тестируемости SEduM на мобильных

## Зачем

| Платформа | Инъекция камеры на уровне ОС |
|---|---|
| Chrome desktop | ✅ видео (Y4M) |
| Android emulator | ⚠️ штатно только статика |
| **iOS Simulator** | ❌ **камеры нет вообще** |

`AVCaptureDevice` в iOS Simulator не возвращает ни одного устройства. Это значит:
**нельзя делать ОС-камеру зависимостью тест-сьюта.** Иначе SEduM на iOS
не покрывается автотестами в принципе.

Решение — инвертировать зависимость: один узкий интерфейс-источник кадров,
три реализации. Тесты подставляют фикстурную.

## Правила

1. `FrameSource` — **единственное** место, импортирующее `AVFoundation` / `androidx.camera`.
2. Фикстурное видео зашито в test/debug-сборку, выбирается launch-аргументом.
3. `FileFrameSource` работает по **виртуальным часам**: 300 кадров прокручиваются
   мгновенно, а не за 10 секунд реального времени.
4. Контрактный тест гоняется против обеих реализаций — чтобы фейк не разошёлся с реальностью.

Выберите файл под свой стек: `FrameSource.kt` · `FrameSource.swift` · `frameSource.ts` (React Native).

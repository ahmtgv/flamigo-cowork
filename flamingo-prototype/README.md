# Flamingo — прототип (DEMO)

Кликабельный прототип браузерной версии Flamingo для раннего этапа: вход по ссылкам-ролям (без авторизации), синтетические данные, **замоканный SEduM** (без реальной камеры/отправки). Стек: React + Vite + TypeScript.

## Локально

```bash
npm install
npm run dev       # http://localhost:5173
npm run build     # прод-сборка в dist/
npm run preview   # предпросмотр сборки на :4173
```

## Роли (демо-вход)

| Ссылка | Кабинет |
|---|---|
| `/r/student` | ученик |
| `/r/teacher` | преподаватель |
| `/r/school` | учебное заведение |
| `/r/parent` | родитель |
| `/r/admin` | администратор |

`/` — экран выбора роли. Роль хранится в `localStorage.demo_role`, бэкенда нет.
Для показов можно форсировать состояние SEduM: `/lesson/1?sedum=away`, `?fatigue=1`, `?cam=deny`.

## ⚠️ Границы прототипа (не нарушать)

- Только **синтетические** данные. Никаких реальных учеников.
- SEduM **не** трогает реальную камеру и **ничего не шлёт** на сервер.
- `noindex` + `robots: Disallow` + `X-Robots-Tag` включены.
- На публичном хосте включить пароль на сайт (Cloudflare Access).
- Ссылки-роли — прототипный обходной путь. В реальном продукте вход возвращается
  (регистрация, согласие родителя по 152-ФЗ, роли через бэкенд).

## Связка с тестами

Набор `../flamingo-tests` в demo-режиме (`authMode: 'demo'`) ходит на `/r/<role>` и гоняет UI-проекты:

```bash
cd ../flamingo-tests
BASE_URL=http://localhost:4173 npm run test:demo
```

Деплой и домен — см. `DEPLOY.md`.
